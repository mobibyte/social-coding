Git cheatsheet

Regularly used commands for day to day operations

Commands

Command Description
git clone [url] Clone a repository
git status Show modified files in directory, staged for commit
git add [path of file] Stages file created/updated
git commit -m "message" Commits the staged file to repository with a message
git push [alias] [branch] Push local changes in the machine to the repository on the internet
git stash Save modified and staged changes
git remote add [name] [url] Adds the url under an alias
git fetch [name] Retrieve changes from a remote repository, specifically from a remote named "[name]"
git branch [branch-name] Creates a branch at current commit
git checkout [branch-name] Switch to another branch and checkout in the working directory
git log Retrieve commit history for active branch in the repository
.gitignore file list of files or directories that will not be part of the repository or commit history


Conventional commits

Commit type Defintion Example
feat Highlights that a feature has been added to an implementation git commit -m "feat: added square root (sqrt) operation to calculator app"
fix Highlights that a previous feature that had a bug has been fixed git commit -m "fix: equal button in calculator app"
docs Presents change in documentation in terms of comments to code, doc strings or changes in readme git commit -m "docs: created docstring highlighting the parameters used in the add function"
chore Indicate changes that are related to tasks or maintenance activities that are not directly related to adding a feature or fixing a bug git commit -m "chore: Update README file with project documentation"
perf Changes that are aimed at improving the performance of the software. Involve optimizations, enhancements, or other modifications designed to make the software run faster, use fewer resources, or improve efficiency. git commit -m "perf: Optimize database query to improve response time"
BREAKING CHANGE / ! Modifications made that impacts the existing functionality. Involves changes that require update in software or code or any configurations that accomodate the new version. git commit -m "BREAKING CHANGE: the add function can work on any number of arguments instead of 2 parameters in the previous version" git commit -m "feat!: Calculating sqrt of number using np.sqrt()"
refactor Improve the existing code readability, maintainability without affecting the functionality git commit -m "refactor: Renaming variables, functions, or classes to better reflect their purpose. "


Thesaurus

Name Defintion
Clone Make a copy of the original
Stage Preparing changes in file or directory to be commited
Commit Freeze changes added/made with a message
Remote Refers to the versions of a repository that are hosted on servers
Branch branch is a parallel version of a repository's codebase that diverges from the main line of development. It allows developers to work on different features, bug fixes, or experiments without affecting the main codebase until they are ready to merge their changes back into it.
